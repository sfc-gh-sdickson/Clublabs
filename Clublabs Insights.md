

"Clublabs Insights \- Technical Overview & Snowflake Integration

1. Project Vision & Goal The Insights project is an internal analytics platform designed to provide deep, actionable insights into the performance and user behavior of Clublabs’s various apps, starting with AAA's Emergency Roadside Service (ERS) chatbot and Online Bind web app. The primary goal is to move from reactive problem-solving to proactive, data-driven optimization of the user journey, thereby improving member satisfaction and service efficiency. This platform will serve as the single aggregation point of Clublabs apps’ operational and performance logs, feeding everything from executive dashboards to an AI-powered agent that provides daily operational digests, conversation analytics interface, and visualization AI- generated charts.  
2. Current Challenges: Our current solution relies on: fragile NLP-to-SQL chatbot interface, lack of low latency, historical context management, manual data acquisition via scripts and ad hoc data source integration. This process is manual, unreliable, and does not provide the real-time, granular data needed to effectively manage and improve the service. This has created a significant capabilities gap that we plan to close with a modern, scalable data architecture.  
3. Proposed Snowflake Architecture: A Scalable ELT Pipeline We are architecting a new, streaming-first pipeline that leverages the core strengths of AWS and Snowflake. The architecture is designed for scalability, maintainability, and near real-time data availability. Data Flow: AWS S3 (us-east-1, us-west-2) → Snowpipe (Auto-Ingest) → Raw Staging Table (VARIANT) → Streams & Tasks → Structured Analytics Tables A few separate data flows to ingest production conversation transcripts and IVR data from non- AWS sources, such as MongoDB and AAA internal systems, might need to be created with help from Snowflake support team. Key Implementation Details: • Source Data for AWS: Application logs are continuously generated by the ERS chatbot, OLB app and backed up from AWS Kinesis to S3 buckets across two regions (us-east-1 and us-west-2) as compressed JSON files. • Ingestion (Load): We will use Snowpipe to automatically and continuously ingest these raw,

compressed JSON logs as they land in S3. Two pipes, one for each region, will feed a single raw staging table, seamlessly merging the data. • Transformation (Transform): Raw data will be loaded into a VARIANT column. A Snowflake Stream will track new records, and a recurring Task will execute a MERGE statement to: 

1. Parse the semi-structured JSON.  
2. Filter for only the relevant ERS chatbot application logs.  
3. Transform and load the clean data into a structured, star-schema model suitable for BI and analytics.

• Analysis & AI/ML: Once the data is structured, we plan to leverage Snowflake's advanced capabilities:  • Business Intelligence: Standard SQL queries will feed our BI dashboards for monitoring KPIs. • AI/ML: We will heavily utilize Snowpark and Snowflake Cortex for sophisticated analysis, including user feedback sentiment analysis, anomaly detection, and natural language queries against our data. 4\. Key Use Cases & Desired Outcomes • Real-time Funnel Analysis: Track user progression through the 19-step chatbot flow to identify drop-off points. The NLP-to-SQL will be exposed to Insights v1.0 app to allow user entering custom questions. The answers produced by Snowflake will also be used by an AI agent to generate visualization. • Sentiment Analysis: Analyze user feedback at scale to quantify member satisfaction and identify emerging issues. • Anomaly Detection: Automatically flag unusual patterns in chatbot errors or user behavior. • AI-Powered Summarization: The final structured data will be consumed by an internal AI agent (insights-bot) to generate daily performance digests for the project team."

\-----**Implementation Plan**

The implementation is structured into four main phases, moving from initial setup to advanced analytics and final solution deployment.**Phase 1: Foundation & Initial Ingestion (S3 Data)**

**Goal:** Establish the core infrastructure to continuously load raw log data from AWS S3 into Snowflake.

* **Setup AWS Infrastructure:**  
  * Provision and secure S3 buckets in both `us-east-1` and `us-west-2` regions.  
  * Configure ERS Chatbot and OLB applications to continuously back up compressed JSON logs from AWS Kinesis to the respective S3 buckets.  
* **Snowflake Setup:**  
  * Create a single **Raw Staging Table** with a dedicated `VARIANT` column to hold the raw JSON.  
  * Configure two separate **Snowpipe (Auto-Ingest)** instances. Each pipe will monitor one of the S3 regions (`us-east-1` and `us-west-2`) and feed the data seamlessly into the single Raw Staging Table.

**Phase 2: Data Transformation and Modeling**

**Goal:** Create a reliable, scheduled process to clean, structure, and model the raw data for analysis.

* **Data Modeling:**  
  * Design the optimal **Star-Schema Model** for the **Structured Analytics Tables**, focusing on key entities (e.g., user journey steps, service requests, conversation metrics).  
* **Transformation Logic:**  
  * Create a **Snowflake Stream** on the Raw Staging Table to track only new records since the last processing run.  
  * Develop a **Snowflake Task** to run on a recurring schedule (e.g., every 5 minutes). This task will execute a **MERGE** statement that:  
    1. Reads the new data from the Stream.  
    2. Applies JSON parsing to extract all necessary fields from the `VARIANT` column.  
    3. Filters for only the relevant ERS chatbot application logs (and potentially other high-priority data).  
    4. Transforms and standardizes the clean data.  
    5. Loads the structured data into the Star-Schema tables.

**Phase 3: Auxiliary Data Integration**

**Goal:** Integrate non-AWS sources for a complete view of the operational data.

* **External Data Ingestion Flows:**  
  * **Consultation:** Engage the **Snowflake Support Team** to architect custom ingestion solutions for production conversation transcripts and IVR data residing in MongoDB and AAA internal systems.  
  * **Development:** Implement the new data flows (e.g., using external staging, API connectors, or other Snowflake-recommended patterns).  
  * **Integration:** Map and integrate this auxiliary data into the existing **Structured Analytics Tables** to enrich the core dataset.

**Phase 4: Advanced Analysis & Outcome Activation**

**Goal:** Leverage Snowflake's advanced capabilities to achieve the desired outcomes and close the capabilities gap.

* **Business Intelligence (BI):**  
  * Connect BI tools to the Structured Analytics Tables.  
  * Develop executive and operational dashboards for monitoring key performance indicators (KPIs) and service health.  
  * Create a real-time funnel analysis dashboard to track the 19-step chatbot flow and identify drop-off points.  
* **AI/ML & Advanced Capabilities:**  
  * Utilize **Snowpark** and **Snowflake Cortex** to build and deploy advanced analytical models:  
    * **Sentiment Analysis:** Analyze user feedback at scale to quantify member satisfaction and detect emerging issues.  
    * **Anomaly Detection:** Automatically flag unusual patterns in chatbot errors or user behavior.  
  * **Natural Language Interface:** Expose the NLP-to-SQL functionality (Insights v1.0 app) for users to enter custom questions, with the answers used by an AI agent to generate visualizations.  
* **AI-Powered Summarization:**  
  * Integrate the internal AI agent (`insights-bot`) to consume the final structured data and automatically generate daily performance digests for the project team.
